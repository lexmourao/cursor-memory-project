# Cursor Operational Rules

These rules are interpreted by the Cursor AI assistant when interacting with this repository.

## 1. Memory & Context

- Maintain a rolling summary of significant interactions in `diary/project_log.md`.
- Before acting, review `PROJECT_RULES.md`, `README.md`, `docs/ARCHITECTURE.md`, and `docs/CONTEXT_ENGINEERING_GUIDE.md`.
- When reading files, retrieve them in chunks of 200 lines or fewer.
- Proactively surface relevant past context to avoid circular work or duplicated effort.
- Treat `memory-bank/` as a starter template until a real project begins.
- Do not fill memory-bank files with fake project knowledge.

## 2. Logging & Status

- Log errors and solutions with `scripts/log_action.py`.
- Update status and checklists with `scripts/update_status.py`.
- Important actions should also be appended to `diary/project_log.md`.

## 3. No-Deletion Policy

- Do not delete project knowledge, documentation, source files, or user-provided data.
- Move outdated project artifacts to `archive-cursor_memory_project/` when preservation is required.
- Generated caches, local indexes, temporary files, and local artifacts may be removed by maintenance commands.

## 4. Folder Consistency

- Preserve the directory structure defined in `PROJECT_RULES.md`.
- New files must live in the appropriate folder.
- If unsure, consult `docs/ARCHITECTURE.md`.

## 5. Security & Compliance

- Follow the guidelines in `docs/SECURITY.md`.
- Do not store API keys, passwords, credentials, personal data, protected health information, client-confidential information, or secrets in `memory-bank/`.
- Sensitive local configuration belongs in `.env`, Docker secrets, or approved secure storage.
- Raw source data, when used, belongs in `data/raw/` and should remain immutable.

## 6. Automation

- Prefer scripted actions over manual editing when the action is repeatable.
- Re-run relevant tests before committing code changes.
- Use `make summarize` or `scripts/summarize_chat.py` intentionally; do not create fake summaries.

## 7. External Context Loader

- On session start, fetch long-term memory from `http://localhost:7331/memory` when the local MCP server is running.
- Retry up to 3 times if the endpoint is not yet available.
- If the MCP server is unavailable, continue in manual/template mode and document the limitation.

---

Adherence to these rules helps the Cursor assistant retain context, avoid circular work, and keep project state transparent and reproducible.
